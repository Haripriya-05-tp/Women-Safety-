# Requirements Document

## Introduction

Women Safety Alert is a lightweight mobile application designed to help women quickly send emergency alerts and share their live GPS location with trusted contacts. The app prioritizes speed and simplicity — a single SOS button triggers SMS alerts to up to 3 pre-configured emergency contacts, including a Google Maps link to the user's current location.

## Glossary

- **App**: The Women Safety Alert mobile application
- **User**: The person using the app who may need emergency assistance
- **Trusted_Contact**: A pre-registered emergency contact (name + phone number) saved by the User
- **SOS_Button**: The primary red emergency button displayed on the home screen
- **Alert_Service**: The component responsible for sending SMS messages to Trusted_Contacts
- **Location_Service**: The component responsible for fetching the device's GPS coordinates and generating a Google Maps link
- **Auth_Service**: The component responsible for user registration and login via phone number or email
- **Contact_Manager**: The component responsible for storing and managing Trusted_Contacts
- **Alarm_Service**: The component responsible for playing a loud alarm sound on the device

## Requirements

### Requirement 1: User Registration and Authentication

**User Story:** As a new user, I want to register and log in with my phone number or email, so that my emergency contacts and settings are saved securely.

#### Acceptance Criteria

1. THE Auth_Service SHALL support user registration using a valid email address and password.
2. THE Auth_Service SHALL support user registration using a valid phone number via OTP verification.
3. WHEN a user submits valid registration credentials, THE Auth_Service SHALL create a new user account in Firebase Authentication.
4. WHEN a user submits invalid or already-registered credentials, THE Auth_Service SHALL display a descriptive error message to the User.
5. WHEN a registered user submits correct login credentials, THE Auth_Service SHALL authenticate the user and navigate to the home screen.
6. IF authentication fails due to incorrect credentials, THEN THE Auth_Service SHALL display an error message without revealing which field is incorrect.
7. WHEN a user is successfully authenticated, THE App SHALL persist the session so the user remains logged in on subsequent app launches.

---

### Requirement 2: Trusted Contacts Management

**User Story:** As a user, I want to add up to 3 emergency contacts with their name and phone number, so that they are notified when I trigger an SOS alert.

#### Acceptance Criteria

1. THE Contact_Manager SHALL allow the User to add a Trusted_Contact by providing a name and a valid phone number.
2. THE Contact_Manager SHALL store a maximum of 3 Trusted_Contacts per user account.
3. WHEN the User attempts to add a fourth Trusted_Contact, THE Contact_Manager SHALL display a message indicating the maximum limit has been reached.
4. THE Contact_Manager SHALL allow the User to delete an existing Trusted_Contact.
5. THE Contact_Manager SHALL persist Trusted_Contacts in Firebase Firestore linked to the authenticated user's account.
6. WHEN the User navigates to the contacts screen, THE Contact_Manager SHALL display all currently saved Trusted_Contacts.
7. IF a phone number entered by the User is not in a valid format, THEN THE Contact_Manager SHALL display a validation error before saving.

---

### Requirement 3: SOS Emergency Alert

**User Story:** As a user in danger, I want to press a single SOS button to instantly notify all my trusted contacts with my location, so that help can reach me quickly.

#### Acceptance Criteria

1. THE App SHALL display the SOS_Button prominently on the home screen as the primary UI element.
2. WHEN the User presses the SOS_Button, THE Alert_Service SHALL send an SMS message to every saved Trusted_Contact.
3. WHEN sending the SMS alert, THE Alert_Service SHALL include the message: "I am in danger. Please help. My location: [Google Maps link]" where [Google Maps link] is generated by the Location_Service.
4. WHEN the User presses the SOS_Button and no Trusted_Contacts are saved, THE App SHALL display a prompt instructing the User to add at least one Trusted_Contact before sending an alert.
5. WHEN the User presses the SOS_Button and location permission is denied, THE Alert_Service SHALL send the SMS alert without a location link and notify the User that location could not be included.
6. IF the SMS sending fails for one or more Trusted_Contacts, THEN THE Alert_Service SHALL display the names of contacts for whom delivery failed.
7. WHEN the SOS alert is successfully sent to all Trusted_Contacts, THE App SHALL display a confirmation message to the User.

---

### Requirement 4: Real-Time Location Fetching

**User Story:** As a user, I want my current GPS location to be automatically included in the SOS alert, so that my trusted contacts know exactly where I am.

#### Acceptance Criteria

1. WHEN the SOS_Button is pressed, THE Location_Service SHALL request the device's current GPS coordinates.
2. WHEN GPS coordinates are obtained, THE Location_Service SHALL generate a Google Maps URL in the format: `https://maps.google.com/?q={latitude},{longitude}`.
3. WHEN the device GPS is unavailable or times out after 10 seconds, THE Location_Service SHALL return a failure status to the Alert_Service.
4. THE Location_Service SHALL request location permission from the User before the SOS_Button is first used.
5. IF the User denies location permission, THEN THE Location_Service SHALL return a permission-denied status to the Alert_Service.

---

### Requirement 5: Alarm Sound (Optional Feature)

**User Story:** As a user, I want a loud alarm to play when I press SOS, so that people nearby are also alerted to my situation.

#### Acceptance Criteria

1. WHERE the alarm feature is enabled, WHEN the User presses the SOS_Button, THE Alarm_Service SHALL play a loud alarm sound on the device at maximum volume.
2. WHERE the alarm feature is enabled, WHEN the alarm is playing, THE App SHALL display a button to stop the alarm.
3. WHERE the alarm feature is enabled, WHEN the User presses the stop button, THE Alarm_Service SHALL immediately stop the alarm sound.
4. WHERE the alarm feature is enabled, THE App SHALL allow the User to enable or disable the alarm feature from the settings screen.
